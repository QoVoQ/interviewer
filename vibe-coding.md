1. 考查“提示词工程”与“任务拆解”能力
提问示例： “在你的 RAG 系统中，如果要增加一个‘多轮对话自动生成医疗报告’的功能，你会如何给 Cursor 或 Claude 写提示词（Prompt）？你会如何拆解这个开发步骤以确保 AI 产出的代码逻辑不出错？”

优秀回答： 能够提到 Chain of Thought (思维链)，先让 AI 输出伪代码/设计逻辑，再输出实现代码；或者提到 上下文注入（比如给 AI 提供现有的接口文档、组件库规范），而不是只发一句“帮我写个报告功能”。

1. 考查“代码审查与纠错”能力（核心）
提问示例： “AI 生成的代码有时会过度渲染（Re-render）或者在处理复杂异步逻辑时产生 Bug。你在项目中使用 Cursor 开发时，有没有遇到过 AI 生成的代码虽然能跑通，但你发现它写得不好并手动重构的情况？请举具体例子。”

1. 考查“Agent 架构理解”能力

提问示例： “你在 RAG 系统中实现的 Agent 指令驱动交互，底层是如何处理‘工具调用（Tool Calling）’失败的情况的？如果大模型给出的 JSON 指令格式错误，你的前端是如何容错并引导用户重新输入的？”

考查点： 是否了解 LLM 的不确定性。优秀的候选人会提到 Schema 验证（如 Zod）、重试机制 或 结构化输出（Structured Outputs） 的处理。


4. 考查对工具局限性的认识
提问示例： “你觉得目前 Cursor 或 Claude Code 在处理大型前端工程时，最明显的‘短板’是什么？在哪些场景下你宁愿自己手写也不愿意依赖 AI？”

考查点： 考查候选人的判断力。如果他说“AI 无所不能”，那说明他可能缺乏深度思考；如果他能提到“跨文件的深度架构耦合”、“极高性能优化”或“复杂的 CSS 微调”是 AI 的弱项，说明他有实战心得。


5. 我假设你的代码有很大一部分是 AI 生成的，这没问题。但我需要你向我证明，每一行 AI 生成的代码都在你的完全掌控之下，并且你能够通过指令引导它写出比普通开发者更优雅的架构。
